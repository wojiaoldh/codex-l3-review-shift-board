# 开发计划

## 阶段 0：范围确认

目标：锁定 L3 作业 App 的 MVP 范围，避免做成过大的系统。

验收标准：

- 原始需求清楚。
- MVP 范围清楚。
- 明确不做登录、云端数据库、权限系统、真实 API。

## 阶段 1：数据结构

目标：设计试点需求记录、指标、证据链和项目 Skills 的数据结构。

涉及字段：

- title
- owner
- stage
- risk
- reviewMinutes
- aiUsed
- aiAdopted
- prdReturned
- prdReturnReason
- shiftedIssues
- deliveryPassed
- evidence
- humanConfirm

验收标准：

- 样例数据能计算 5 个指标。
- 每条记录至少有证据和人工确认点。

## 阶段 2：指标看板

目标：展示 5 个核心指标。

指标：

- 平均评审时长
- AI 报告采纳率
- PRD 打回率
- 质量左移率
- 交付评审通过率

验收标准：

- 指标随数据变化自动更新。
- 指标卡在桌面和移动宽度下不重叠。

## 阶段 3：记录列表和筛选

目标：展示试点需求记录，并支持风险、阶段和关键词筛选。

验收标准：

- 能看到需求名称、负责人、阶段、风险。
- 风险筛选生效。
- 阶段筛选生效。
- 关键词搜索生效。

## 阶段 4：详情和证据链

目标：点击记录后查看指标口径、证据链和人工确认点。

验收标准：

- 点击记录后右侧详情更新。
- 证据链和人工确认点清楚展示。
- 可标记人工确认。

## 阶段 5：新增记录和本地存储

目标：支持手动新增试点需求记录，并保存在 localStorage。

验收标准：

- 表单必填项可保存。
- 新记录出现在列表顶部。
- 刷新页面后记录仍存在。

## 阶段 6：导出和项目 Skills

目标：支持导出周同步 Markdown，并沉淀项目 Skills。

验收标准：

- 导出按钮复制 Markdown。
- 项目 Skills 在 App 中展示。
- `project-skills` 下有 3 个可复用 Skill。

## 阶段 7：验收修复

目标：检查功能完整性、样式、响应式和项目材料。

验收标准：

- 本地服务可启动。
- 浏览器验证通过。
- Skill 校验通过。
- Git 仓库初始化完成。
