# Codex 训练营 L3 作业提交说明

## 推荐提交标题

用 Codex 做一个真实可用的评审虾质量左移指标看板

## 推荐粘贴到作业表单的摘要

本次 L3 作业选择了我的真实 OKR 主线：评审虾质量左移量化体系。  

我没有做通用 Todo 或玩具 Demo，而是用 Codex 做了一个 PC 网页端 App：`评审虾质量左移指标看板`。它可以记录试点需求、计算评审时长、AI 报告采纳率、PRD 打回率、质量左移率、交付评审通过率，并为每条记录维护证据链和人工确认点，还可以导出周同步 Markdown。

项目按 L3 方法完成：先写原始需求，再定义 MVP，再拆开发计划，再用 Codex + Superpowers 实现 App，再做浏览器验收修复，最后沉淀 3 个项目 Skills：`review-metric-recording`、`ai-report-adoption-check`、`left-shift-evidence-map`。
