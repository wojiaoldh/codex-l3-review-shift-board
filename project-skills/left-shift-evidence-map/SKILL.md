# Left Shift Evidence Map

## Purpose

Make every quality-left-shift conclusion traceable to review evidence.

## Inputs

- Requirement record
- Review notes
- AI report suggestions
- PRD return reason
- Delivery review result
- Manual confirmation notes

## Process

1. Write the quality-left-shift claim in one sentence.
2. List evidence that directly supports the claim.
3. Label evidence strength as high, medium, or low.
4. Separate fact, inference, and pending confirmation.
5. Identify missing evidence before leader-facing reporting.

## Output

Use this table:

| Claim | Evidence | Source | Strength | Human confirmation |
|---|---|---|---|---|

## Verification

- Every claim has at least one source.
- Low-strength evidence is not used as final proof.
- Pending confirmation is visible.
- Internal links, project names, and sensitive data are removed before external sharing.
